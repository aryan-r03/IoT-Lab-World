## ⚡ Quick Start

<div align="center">

### Step 1️⃣: Clone the Repository

</div>

```bash
git clone https://github.com/aryan-r03/IoT-Lab-World.git
cd IoT-Lab-World
```

<div align="center">

### Step 2️⃣: Install Arduino IDE

</div>

1. Download from [arduino.cc](https://www.arduino.cc/en/software)
2. Install for your OS (Windows / macOS / Linux)
3. Launch Arduino IDE

<div align="center">

### Step 3️⃣: Add ESP32 Board Support (for ESP32 projects)

</div>

1. Open **File → Preferences**
2. Paste in "Additional boards manager URLs":
   ```
   https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json
   ```
3. Go to **Tools → Board → Boards Manager**
4. Search **"esp32"** and install

<div align="center">

### Step 4️⃣: Install Required Libraries

</div>

Go to **Sketch → Include Library → Manage Libraries** and install:

```
DHT sensor library       → by Adafruit
Adafruit Unified Sensor  → by Adafruit
NewPing                  → by Tim Eckel
LiquidCrystal I2C        → by Frank de Brabander
Adafruit SSD1306         → by Adafruit
```

<div align="center">

### Step 5️⃣: Open a Project & Upload

</div>

```
IoT-Lab-World/
└── 01_Ultrasonic_Distance/
    └── ultrasonic_distance.ino   ← Open this in Arduino IDE
```

## ▶️ How to Run

<div align="center">

### Full Upload Workflow

</div>

```
┌─────────────────────────────────────────────────────┐
│              ARDUINO IDE WORKFLOW                   │
├─────────────────────────────────────────────────────┤
│                                                     │
│  1. Open .ino file                                  │
│     File → Open → project_folder/project.ino        │
│                                                     │
│  2. Select Board                                    │
│     Tools → Board → Arduino Uno                     │
│                       OR                            │
│                     ESP32 Dev Module                │
│                                                     │
│  3. Select Port                                     │
│     Tools → Port → COM3 (Windows)                   │
│                    /dev/ttyUSB0 (Linux)             │
│                    /dev/cu.usbserial (macOS)        │
│                                                     │
│  4. Verify Code                                     │
│     Click ✓ (Verify button) or Ctrl+R               │
│                                                     │
│  5. Upload                                          │
│     Click → (Upload button) or Ctrl+U               │
│                                                     │
│  6. Monitor Output                                  │
│     Tools → Serial Monitor (Ctrl+Shift+M)           │
│     Set baud rate: 9600 or 115200                   │
│                                                     │
└─────────────────────────────────────────────────────┘
```

## 🔌 Hardware Setup

<div align="center">

### Pin Reference — Arduino Uno

</div>

```
Arduino Uno Pinout
┌────────────────────────────────────────────┐
│  DIGITAL PINS  │  0–13 (0,1 = UART)        │
│  ANALOG PINS   │  A0–A5                    │
│  PWM PINS      │  3, 5, 6, 9, 10, 11       │
│  I2C           │  SDA=A4, SCL=A5           │
│  SPI           │  MOSI=11, MISO=12, SCK=13 │
│  POWER         │  5V, 3.3V, GND            │
│  OPERATING V   │  5V logic                 │
└────────────────────────────────────────────┘

HC-SR04 (Ultrasonic) → Arduino Uno
  VCC  → 5V
  GND  → GND
  TRIG → Digital Pin 9
  ECHO → Digital Pin 10

DHT11 (Temp/Humidity) → Arduino Uno
  VCC  → 5V
  GND  → GND
  DATA → Digital Pin 2 (with 10kΩ pull-up)

Servo Motor → Arduino Uno
  Red   → 5V
  Brown → GND
  Orange → Digital Pin 9 (PWM)

I2C LCD (16x2) → Arduino Uno
  VCC → 5V
  GND → GND
  SDA → A4
  SCL → A5
```

<div align="center">

### Pin Reference — ESP32

</div>

```
ESP32 DevKit Pinout (Key Pins)
┌────────────────────────────────────────────┐
│  GPIO PINS     │  0–39 (many with special  │
│                │  functions)               │
│  ADC PINS      │  32–39, 0, 2, 4, 12–15    │
│  DAC PINS      │  25, 26                   │
│  I2C           │  SDA=21, SCL=22           │
│  SPI           │  MOSI=23, MISO=19, SCK=18 │
│  UART          │  TX=1, RX=3               │
│  POWER         │  3.3V, 5V (Vin), GND      │
│  OPERATING V   │  3.3V logic!              │
└────────────────────────────────────────────┘

HC-SR04 → ESP32
  VCC  → 5V (Vin)
  GND  → GND
  TRIG → GPIO 5
  ECHO → GPIO 18 (use voltage divider!)

⚠️  ESP32 is 3.3V logic — use voltage dividers
    or logic level shifters for 5V sensors!
```

## 📡 Arduino vs ESP32

<div align="center">

### Which Board to Use?

</div>

<table>
  <tr>
    <th>Feature</th>
    <th>Arduino Uno</th>
    <th>ESP32</th>
  </tr>
  <tr>
    <td><b>Processor</b></td>
    <td>ATmega328P (8-bit, 16MHz)</td>
    <td>Dual-core Xtensa LX6 (32-bit, 240MHz)</td>
  </tr>
  <tr>
    <td><b>Flash Memory</b></td>
    <td>32 KB</td>
    <td>4 MB</td>
  </tr>
  <tr>
    <td><b>RAM</b></td>
    <td>2 KB SRAM</td>
    <td>520 KB SRAM</td>
  </tr>
  <tr>
    <td><b>WiFi</b></td>
    <td>❌ No</td>
    <td>✅ Built-in 802.11 b/g/n</td>
  </tr>
  <tr>
    <td><b>Bluetooth</b></td>
    <td>❌ No</td>
    <td>✅ BLE + Classic</td>
  </tr>
  <tr>
    <td><b>Operating Voltage</b></td>
    <td>5V logic</td>
    <td>3.3V logic</td>
  </tr>
  <tr>
    <td><b>Digital I/O Pins</b></td>
    <td>14</td>
    <td>34</td>
  </tr>
  <tr>
    <td><b>Analog Input Pins</b></td>
    <td>6</td>
    <td>18</td>
  </tr>
  <tr>
    <td><b>Best For</b></td>
    <td>Simple sensors, learning, offline projects</td>
    <td>IoT, web dashboards, cloud, data-heavy apps</td>
  </tr>
  <tr>
    <td><b>Cost</b></td>
    <td>~₹350 - 600</td>
    <td>~₹450 - 700</td>
  </tr>
</table>

**Rule of thumb:**
- 🔵 Need **WiFi or Bluetooth**? → Use **ESP32**
- 🟢 Simple **sensor + output**? → Use **Arduino Uno**
- 🟡 Need **more processing power**? → Use **ESP32**
